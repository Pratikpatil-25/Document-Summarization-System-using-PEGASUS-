With the emergence of machine learning, researchers began applying data-driven models to understand programming patterns and predict code quality. Supervised learning techniques have been used to classify code submissions, detect bugs, and estimate performance metrics such as time and space complexity. These approaches offer improved flexibility over rule-based systems; however, they often require large labeled datasets and may lack interpretability, making it difficult for learners to understand the reasoning behind the evaluation. More recently, the development of large language models has significantly influenced the domain of code analysis and education. Generative AI models are capable of understanding code context and producing human-like explanations, suggestions, and even optimized code snippets. These models enhance the learning experience by providing descriptive feedback and conceptual clarity. Nevertheless, generative models alone may not always guarantee precise analytical accuracy, particularly in tasks such as complexity estimation and structured code evaluation. To overcome the individual limitations of analytical and generative approaches, recent studies have explored hybrid AI systems that combine multiple models to leverage their complementary strengths. Such systems aim to integrate the precision of fine-tuned models with the interpretability of generative models. However, the application of hybrid AI specifically in educational code assessment platforms remains relatively underexplored, especially in terms of delivering both accurate technical evaluation and meaningful pedagogical feedback within a unified system. Based on the identified gaps, this paper proposes a hybrid AI-based educational code assessment platform that combines fine-tuned and generative models to provide comprehensive evaluation and enhanced learning support. The proposed approach seeks to contribute to the advancement of intelligent educational tools by bridging the gap between automated assessment and effective pedagogy. III. PROPOSED SYSTEM This section describes the design and working of the proposed educational code assessment platform, which utilizes a hybrid artificial intelligence approach to provide both accurate evaluation and meaningful learning support. The methodology integrates automated code execution with AI-driven analysis to assess not only the correctness of a program but also its efficiency, structure, and conceptual quality. 3.1 System Overview The proposed system follows a multi-stage pipeline where user- submitted code is processed through execution, analysis, and feedback generation modules. Initially, the user submits code through the platform interface. The code is then compiled and executed in a controlled environment to verify correctness using predefined test cases. Once execution is complete, the code is forwarded to the hybrid AI module for deeper analysis. The hybrid AI module is the core component of the system, designed to combine the strengths of fine-tuned and generative models. This integration enables the system to deliver both precise technical insights and human-readable explanations, thereby supporting educational objectives. 3.2 Hybrid AI Framework The proposed methodology employs two complementary AI components: 1. Fine-Tuned Analytical Model: A fine-tuned CodeLlama 7B model is used to analyze the structural and computational aspects of the submitted code. This model is trained to estimate time complexity, space complexity, and identify key code patterns. It provides structured outputs that reflect the efficiency and organization of the program. The use of a fine-tuned model ensures higher accuracy and consistency in analytical evaluation. 2. Generative AI Model: A generative AI model, accessed through the Gemini API, is used to generate detailed explanations, optimization suggestions, and conceptual guidance. This component interprets the code in a human-like manner and produces feedback that is easy for students to understand. It focuses on improving code readability, logic clarity, and learning outcomes. The outputs from both models are combined to produce a comprehensive evaluation report. The analytical model ensures technical correctness and precision, while the generative model enhances interpretability and educational value. 3.3 Workflow of the System The overall workflow of the proposed system can be summarized as follows: 1. Code Submission: The user submits a program through the platform interface. 2. Compilation and Execution: The system compiles and executes the code against predefined test cases to verify correctness. 3. Preprocessing: The code is cleaned and formatted to ensure compatibility with AI models. 4. AI-Based Analysis: -The fine-tuned model evaluates time complexity, space complexity, and code structure. -The generative model generates explanations and optimization suggestions. 5. Result Aggregation: Outputs from both models are combined into a unified feedback report. 6. Feedback Delivery: The system presents execution results, complexity analysis, and AI-generated suggestions to the user. 3.4 Educational Features Integration In addition to AI-based analysis, the platform incorporates several features aimed at enhancing the learning experience.
