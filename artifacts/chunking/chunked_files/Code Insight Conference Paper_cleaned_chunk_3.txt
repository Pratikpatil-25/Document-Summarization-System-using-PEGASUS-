SYSTEM ARCHITECTURE system architecture of the proposed platform is designed to ensure efficient processing of user submissions while integrating hybrid AI capabilities for comprehensive code evaluation and feedback. The architecture follows a modular and layered design, allowing scalability, maintainability, and seamless interaction between different components. 4.1 Architectural Overview : The platform is structured into four primary layers: the User Interface Layer, Application Layer, Execution Layer, and AI Analysis Layer, supported by a centralized Database Layer. Each layer performs a specific function in the overall workflow, enabling smooth data flow from code submission to feedback generation. 4.2 User Interface Layer : The User Interface (UI) layer serves as the interaction point between users and the system. It is designed to provide an intuitive and responsive environment where users can write, edit, and submit code. The interface supports features such as problem statements, code editor integration, submission history, and leaderboard visualization. This layer communicates with the backend through API requests, ensuring real-time interaction and feedback display. 4.3 Application Layer : The Application Layer acts as the central controller of the system. It handles user requests, manages session data, and coordinates communication between different modules. When a user submits code, this layer validates the input, forwards it for execution, and later routes it to the AI analysis components. It also aggregates results from various modules and formats them into a structured response for the user interface. 4.4 Execution Layer : The Execution Layer is responsible for compiling and running the submitted code in a secure and isolated environment. It uses predefined test cases to verify the correctness of the program. This layer ensures that code execution is performed safely, preventing unauthorized system access or malicious operations. The output generated from this stage includes execution results, runtime behavior, and error messages, which are essential for initial evaluation. 4.5 AI Analysis Layer : The AI Analysis Layer is the core of the proposed architecture, implementing the hybrid AI approach. It consists of two main components: • Analytical Model Component: This component uses a fine-tuned CodeLlama 7B model to analyze the submitted code. It evaluates time complexity, space complexity, and structural characteristics, producing structured outputs for precise technical assessment. • Generative Model Component: This component utilizes a generative AI model (Gemini API) to generate detailed explanations, optimization suggestions, and conceptual insights. It transforms technical analysis into human-readable feedback, enhancing the learning experience. The outputs from both components are combined within this layer to create a comprehensive evaluation report. Fig. 1. Flowchart if the Workflow of the System. 4.6 Database Layer : The Database Layer stores all essential data, including user information, problem statements, code submissions, execution results, and analysis reports. It also maintains leaderboard data and historical performance records. This persistent storage enables tracking of user progress and supports analytics for system improvement. 4.7 Data Flow : The overall data flow in the system follows a sequential process. The user submits code through the interface, which is processed by the application layer and sent to the execution layer for validation. After execution, the code and results are forwarded to the AI analysis layer for detailed evaluation. Finally, the aggregated feedback is stored in the database and displayed to the user through the interface. 4.8 Architectural Benefits : The modular design of the system architecture offers several advantages: • Scalability: Independent components allow easy expansion of system capabilities • Security: Isolated execution environment ensures safe code processing • Flexibility: Hybrid AI components can be updated or replaced without affecting other modules • Efficiency: Parallel processing of execution and analysis improves response time V.
